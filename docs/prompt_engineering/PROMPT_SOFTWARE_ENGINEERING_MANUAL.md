# Prompt Software Engineering Manual

## Prompt Structure (Recommended)

- Role / mode
- Task
- Context (files, constraints, environment)
- Requirements / acceptance criteria
- Safety boundaries
- Output format
- Verification steps

## Prompting Patterns Covered

- implementation prompt
- debugging prompt
- code review prompt
- test generation prompt
- refactor prompt
- documentation prompt
- migration prompt
- incident response prompt

## Anti-Patterns

- "fix this" with no files/trace
- asking for code without constraints
- mixing brainstorming and production execution in one prompt
- no test or validation request
