# AI-Assisted Software Engineering Lesson Research & Analysis Companion

Author: Simon Parris + Codex companion notes  
Date: 2026-02-24

This companion explains the theory behind each module in beginner-friendly language while preserving technical rigor.

## Analysis Lens For Every Lesson

- What problem class is this solving?
- What assumptions does the approach make?
- What can fail technically?
- What can fail organizationally?
- What evidence proves success?
- What metrics prevent self-deception?

## Module Questions

### Foundations

- What is the difference between next-token prediction and verified software execution?
- Why does context selection matter more than prompt style in many coding tasks?

### Prompt Engineering

- Which prompt fields most reduce ambiguity?
- When does adding more instructions decrease performance?

### Agent Engineering

- What decisions should remain human-approved?
- Which tools are safe by default and which need escalation?

### Evals

- What quality dimensions are measurable vs subjective?
- How do you prevent optimizing prompts only for the benchmark set?

### Security/Governance

- What data should never be sent to external models?
- How do you audit AI-generated changes for license/IP risk?
