# Core Concepts

## AI-Assisted Software Engineering

Using LLM systems to accelerate software delivery while preserving engineering controls, reproducibility, and quality gates.

## Prompt Software Engineering

Designing prompts as testable task specifications with explicit inputs, constraints, and acceptance criteria.

## Context Engineering

Selecting and structuring the right files, requirements, and state for the model so outputs are relevant and safe.

## Agent Engineering

Building systems that combine an LLM with tools, state, policies, and execution loops to complete tasks.

## Eval (Evaluation)

A repeatable check that measures whether a prompt/agent output meets quality expectations.

## Hallucination (Engineering Context)

Confidently generated content not supported by the codebase, tools, or validated facts.

## Guardrail

A rule or control limiting unsafe actions, data exposure, or unsupported output behavior.

## Human-in-the-Loop

A workflow where a person authorizes, reviews, or corrects high-risk decisions and changes.
